Expand description

A wire protocol between a client and a server. The associated types pin down what a Client<Self> wraps and what a Server<Self> wraps; they can (and usually do) differ, since a client holds a connected stream while a server holds a listener.

Protocols with no client half (none today) or no server half (Embedded) use Never for the missing side; the corresponding Client<P> / Server<P> then has no inhabitants and no constructor.

Required Associated Types§

Source

type Client

Per-protocol state carried by Client<Self>. For wire protocols this is typically a connected stream behind a mutex; for Embedded it is the in-proc DB handle and unlock cache.

Source

type Server

Per-protocol state carried by Server<Self>. For wire protocols this is typically a listener; for protocols without a server half (e.g. Embedded) this is Never.
