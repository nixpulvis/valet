IPC message types: Request, Response, the expect_* reply helpers, and the Error / ResponseError shapes typed handlers return. The wire codec for these types lives in super::frame.

Structs§

RevisionEntry
One historical revision of a record, as carried in Response::History. A wire-friendly subset of the in-process crate::record::Revision (no storgit::CommitId, no SystemTime baggage).

Enums§

Error
RPC-layer error for valet handler clients. T is the transport-specific failure type: std::io::Error for the socket handler and FFI, a richer enum for the browser extension’s native-messaging path.
Request
A message sent from a client to the handler. Each variant is answered by exactly one Response (possibly Response::Error).
Response
A message sent from the handler back to the client in reply to a Request. Every request produces exactly one response; Error may be returned in place of any success variant.
ResponseError
Failure variants produced when extracting an expected Response variant via the Response::expect_* methods. Converts into Error for any T via ?.

Functions§

domain_matches
Return true if a domain label matches a query host, using the loose suffix rule that powers the Request::FindRecords RPC. The match is symmetric across the dot boundary: "sub.example.com" matches "example.com" and vice-versa; "example.com" does not match "other.com".
label_matches_domain
Return true if label is a domain label matching query under domain_matches.
