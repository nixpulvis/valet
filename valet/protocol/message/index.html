Module message

IPC message types. Request and Response are the wire enums; Call plus one payload struct per variant drive Handler::call; ResponseError covers the reply-inspection failures Call::from_response can raise. The wire codec for Request / Response lives in super::frame.

Enums§

Request
A message sent from a client to the handler. Each variant is answered by exactly one Response (possibly Response::Error).
Response
A message sent from the handler back to the client in reply to a Request. Every request produces exactly one response; Error may be returned in place of any success variant.
ResponseError
Failure variants produced when extracting an expected Response variant via the internal Response::expect_* methods. Converts into crate::protocol::Error via ? inside Call::from_response implementations.

Traits§

Call
One typed RPC call: a payload struct that knows how to become a Request and how to extract its declared Call::Response out of a matching Response variant.

Functions§

domain_matches
Return true if a domain label matches a query host, using the loose suffix rule that powers the Request::FindRecords RPC. The match is symmetric across the dot boundary: "sub.example.com" matches "example.com" and vice-versa; "example.com" does not match "other.com".
label_matches_domain
Return true if label is a domain label matching query under domain_matches.
