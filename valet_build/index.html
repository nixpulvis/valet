Shared build-script helpers.

generate_header drives cbindgen for any workspace crate that exposes a C-ABI surface. All settings common to every FFI header live in cbindgen.toml alongside this crate; crate-specific bits (include guard, preprocessor defines derived from Cargo features) are applied below.

The header lands in two places:

  • target/<triple>/<profile>/include/<name>.h - profile-specific; what a dependent Rust crate would see via the emitted cargo:include=....
  • target/<triple>/include/<name>.h - profile-agnostic; what the macOS extension’s Swift modulemap points at so one fixed path works for both debug and release builds.
