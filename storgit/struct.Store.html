Struct Store

pub struct Store<L: Layout = SubmoduleLayout> { /* private fields */ }
Expand description

Git backed database with a specific layout.

The Layout selects the on-disk layout, which defaults to SubmoduleLayout.

Source

pub fn with_parts(self, parts: Parts) -> Result<Self, Error>

Fold Parts into an opened submodule-layout store.

  • parts.parent: if non-empty, untarred into parent.git, replacing whatever open produced. Requires the existing parent to be empty (freshly initialised). TODO: once merging lands (see PLAN-MERGE.md), allow applying parent bytes onto an existing non-empty parent by merging.
  • parts.modules: inserted into the pending-modules cache; each id’s tarball is untarred on first touch.
Source

pub fn with_fetcher(self, fetcher: ModuleFetcher) -> Self

Install a ModuleFetcher as a lazy fallback for module bytes, replacing any prior fetcher. See ModuleFetcher for the contract.

Source

pub fn snapshot(&mut self) -> Result<Snapshot, Error>

Re-tar everything touched since the previous snapshot (or since Store::<SubmoduleLayout>::open for the first call) and hand the caller exactly the parts that need repersisting. Clears dirty tracking on success, so back-to-back snapshots with no intervening writes return an empty Snapshot.

Source

pub fn load_module(&mut self, id: EntryId, bytes: Vec<u8>)

Make bytes (a previously-persisted module tarball) available to the store under id. The next operation that touches id will untar these bytes into the store’s path on first access.
