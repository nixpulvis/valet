Crate storgit 

Source
Expand description

Labeled, versioned entry storage backed by git.

A storgit Store creates a git repository to use as a database for labeled data. The store provides a thin wrapper around the Layout interface, as well as layout specific constructors and storage methods.

Each entry is keyed by an opaque EntryId and carries two optional payloads inside every commit: a label (cached metadata the caller wants to scan without reading from git) and a data blob (the actual record). The store’s put method writes a commit and updates the entry’s ref, get returns the latest Entry, and history walks the entry’s commits. More information can be found on the Layout trait, or each implementation’s docs.

storgit is payload-agnostic: it stores raw bytes. Encryption, id policy, label format, and any higher-level semantics belong to the caller.
