Module layout 

Source
Expand description

On-disk shapes that back a Store.

The Layout trait is the surface API that every concrete storgit layout (submodule, subdir) implements. A Store is a thin generic wrapper over a layout and just delegates each method to the layout. Available layouts are:

  • SubdirLayout: a single bare repo at <path>/ whose tree carries all entries as <path>/records/<id>/{data,label}, with one shared ref and per-entry history via path-scoped walks.
  • SubmoduleLayout: a <path>/parent.git/ bare repo whose tree carries one gitlink per live entry plus a .gitmodules manifest, alongside <path>/modules/<id>/{data,label} bare repos each holding that entry’s {data,label} blobs in their own object database.

Operations that are specific to one layout (persistence envelopes like submodule::Parts / submodule::Snapshot, etc.) stay as inherent methods on that layout’s Store<L> rather than living on this trait.
